# Firebase 料金まとめ（PiyopiyoStoryBoardMaker）

> 最終更新: 2026年4月

---

## このプロジェクトで使っているサービス

| サービス | 用途 |
|---|---|
| **Authentication** | Googleログイン |
| **Firestore** | ストーリーボードデータの保存 |
| **Cloud Storage** | 画像ファイルの保存・配信 |

---

## プランの種類

Firebase には2つのプランしかない。

| | Spark（無料） | Blaze（従量課金） |
|---|---|---|
| 料金 | 無料 | 使った分だけ |
| クレカ登録 | 不要 | 必要 |
| 無料枠 | あり | Sparkと同じ無料枠が含まれる |

**⚠️ このプロジェクトはBlaze必須**

`VITE_FIREBASE_STORAGE_BUCKET` が `*.firebasestorage.app`（2024年10月30日以降に作成された新形式バケット）のため、Sparkプランでは使用不可。Blazeプランへのアップグレードが必要。

## 画像枚数と料金の目安

画像1枚を平均 **500KB**（AIイラスト・縮小済みJPEG想定）として計算。

### ストレージ料金（保存コスト）

| 保存枚数 | 合計容量 | 月額 |
|---|---|---|
| 1,000枚 | 約 0.5 GB | **無料** |
| 5,000枚 | 約 2.5 GB | **無料** |
| 10,000枚 | 約 5 GB | **無料**（ギリギリ） |
| 20,000枚 | 約 10 GB | 約 **$0.13 / 月**（約20円） |
| 50,000枚 | 約 25 GB | 約 **$0.52 / 月**（約80円） |
| 100,000枚 | 約 50 GB | 約 **$1.17 / 月**（約180円） |

### ダウンロード料金（表示コスト）

画像を表示するたびに転送量がかかる。閲覧頻度によって大きく変わる。

| 月間閲覧回数（全ユーザー合計） | 転送量 | 月額 |
|---|---|---|
| 10,000回 | 約 5 GB | **無料** |
| 100,000回 | 約 50 GB | **無料** |
| 200,000回 | 約 100 GB | **無料**（無料枠の上限） |
| 500,000回 | 約 250 GB | 約 **$18 / 月**（約2,800円） |
| 1,000,000回 | 約 500 GB | 約 **$48 / 月**（約7,500円） |

> ダウンロード料金は **ストレージよりも高い**。画像を大量に表示するアプリは閲覧数がコストの主要因になる。

## 現実的なシナリオ別コスト試算

### ケース① 個人利用・小規模（〜100人）

- ユーザー数: 〜100 MAU
- 画像: 1,000〜3,000枚
- 閲覧: 月10,000〜50,000回

**→ 月額 $0（完全無料）**

### ケース② 中規模（〜1,000人）

- ユーザー数: 1,000 MAU
- 画像: 10,000〜30,000枚
- Firestore: 100万読み取り / 月
- 閲覧: 月200,000〜500,000回

**→ 月額 $1〜$20（約150〜3,000円）**

### ケース③ 大規模（10,000人〜）

- ユーザー数: 10,000 MAU
- 画像: 100,000枚以上
- Firestore: 1,500万読み取り / 月

**→ 月額 $100〜$500+（約15,000〜75,000円）**

## コストを抑えるためのポイント

1. **画像を圧縮してアップロード**  
   Storage容量とダウンロード転送量の両方を削減できる。クライアント側でリサイズ・圧縮してから保存するのが効果的。

2. **画像をキャッシュする**  
   ブラウザキャッシュやCDNを活用して同じ画像の再ダウンロードを防ぐ。FirebaseのURLは長期キャッシュが効く。

3. **Firestoreの読み取りを減らす**  
   ページネーション、ローカルキャッシュ（オフライン永続化）を活用する。このプロジェクトはすでに40MBキャッシュを設定済み（`cacheSizeBytes`）。

4. **予算アラートを必ず設定する**  
   Firebaseには自動課金停止機能がない。Google Cloud Consoleの「予算とアラート」で上限額を設定してメール通知を受け取ること。  
   → [Google Cloud Console > 予算とアラート](https://console.cloud.google.com/billing/budgets)

## まとめ

| 局面 | 月額目安 |
|---|---|
| 個人・プロトタイプ（〜100人） | **$0（無料）** |
| 小〜中規模（〜1,000人） | **$1〜$20** |
| 中〜大規模（〜10,000人） | **$20〜$200** |
| 大規模（10,000人〜） | **$200〜** |

**画像枚数よりも「閲覧（ダウンロード）頻度」の方がコストに直結する**。保存は安いが、表示するたびに転送量がかかる点を意識して設計すること。
